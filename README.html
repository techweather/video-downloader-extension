What it does

dlwithit is a two-part tool: a browser extension (Firefox and Chrome) that detects media on any webpage, and a native macOS app that handles the actual downloading. Right-click on any page to pick images, grab videos from platforms like YouTube and Vimeo, or extract embedded video files that browsers can't normally save.

The app queues downloads with progress tracking, auto-encodes VP9 to H.264, and can embed source URLs as metadata so you always know where something came from.

Requirements

  • macOS 14.4 (Sonoma) or later — Apple Silicon or Intel
  • Firefox or Chrome browser

Everything else — ffmpeg, yt-dlp, exiftool — is bundled. Nothing else to install.

Installing the App

  1. Drag dlwithit.app to your Applications folder.
  2. First launch: Right-click the app → Open. macOS will warn that the app is not notarized — click Open in the dialog to proceed. You only need to do this once.
  3. Grant permission to access your Downloads folder when prompted.
  4. The dlwithit icon will appear in your menu bar.

Installing the Browser Extension

Firefox

Requires Firefox 140 or later. To check your version: Help → About Firefox.
  1. Open the Firefox Extension folder.
  2. Drag dlwithit-extension-1.0.0.xpi onto a Firefox window.
  3. Click Add when prompted.
  4. The extension is now installed. Access dlwithit by right-clicking on any webpage.

Chrome

  1. Open Chrome and go to chrome://extensions.
  2. Enable Developer mode (toggle in the top-right corner).
  3. Click Load unpacked and select the Chrome Extension folder.
  4. The extension is now installed. Access dlwithit by right-clicking on any webpage.

Features

Pick Images

Visual overlay to select and download images from any page.

▶️ Video Download

Download from YouTube, Vimeo, Instagram, TikTok, and hundreds more via yt-dlp; also scans pages for embedded video files (Mux, Squarespace, HLS streams, etc.).

Metadata embedding

Saves the source URL inside downloaded files so you always know where something came from.

VP9 to H.264 encoding

Auto-converts WebM/VP9 downloads to H.264 MP4.

Playlist support

Detects playlists and lets you pick which videos to download.

Organize by platform

Auto-sorts downloads into folders by source site.

Auto-updates yt-dlp

Checks for updates and installs with one click.

First Use

Once the app is running and the extension is installed, right-click any webpage in Firefox to see the dlwithit menu. Downloads appear in the app's queue with progress bars, thumbnails, and status updates. Completed downloads have a "Show in Finder" button.

Downloads are saved to ~/Downloads/dlwithit/, organized by site.

Settings

  • Save location Default ~/Downloads/dlwithit/, or set a custom folder
  • Organize by platform Auto-create subfolders for YouTube, Instagram, etc.
  • Metadata None, embedded in file, or sidecar — see below
  • Auto-encode VP9 Convert WebM/VP9 to H.264 MP4 after download
  • Keep original Retain the original file after encoding
  • System tray Show/hide tray icon, minimize to tray on close

Metadata

When downloading images, dlwithit can save information about where the image came from. Choose an option in the app's settings:

None

No metadata is written. The file is saved exactly as downloaded.

Embedded

Source URL, page title, and date are written directly into the image file using standard EXIF/IPTC fields. The file looks and works normally — the metadata is just hidden inside it.

To view it: open the image in Preview → Tools → Show Inspector, or use Adobe Bridge where it appears in the sidebar under the Description field.

Sidecar

Metadata is saved to a separate .xmp file alongside the image (e.g. photo.jpg + photo.xmp). The sidecar may also include a thumbnail and other details depending on the content. Useful if you don't want to modify the original file.

Troubleshooting

"Add-on appears to be corrupt" when installing extension

This usually means your Firefox version is too old. dlwithit requires Firefox 140 or later. Update Firefox via Help → About Firefox, or download the latest version from mozilla.org.

"App can't be opened because it is from an unidentified developer"

Right-click the app → Open, then click Open in the dialog. You only need to do this once.

Right-click → Open doesn't work (Gatekeeper)

On some Macs, you may need to go to System Settings → Privacy & Security, scroll down to find the message about dlwithit, and click "Open Anyway".

Extension doesn't respond / nothing happens

The dlwithit app must be running for the extension to work. Check your menu bar — if the icon isn't there, launch the app from your Applications folder first.

YouTube downloads failing?

yt-dlp may need an update. If one is available, you'll see an "Update available" link in the app settings. YouTube frequently changes its API and yt-dlp updates are the fix.

403 Forbidden errors on images

Some CDNs reject requests with unexpected headers. dlwithit uses a minimal-headers strategy that works with most CDNs, but some sites may still block downloads.

Videos not detected on a page

▶️ Video Download scans the page source for embedded video URLs in scripts, data attributes, and metadata. Some sites lazy-load video — scroll to the video and let it start playing for a few seconds, then invoke ▶️ Video Download again. If it still doesn't appear, the URL may be loaded dynamically after user interaction and may not be capturable.

Downloaded video won't play or is corrupted

Some sites lazy-load videos — the file only fully loads after you scroll to it and let it play. Go back to the page, play the video for a few seconds, then use ▶️ Video Download again. This is especially common on sites like Apple.com.

Report Issue

If a download fails, the Report Issue button sends information about the URL and error to help improve dlwithit's compatibility with more sites. This helps identify which websites need better support.

No personal information is collected — only the URL and technical error details are sent.
